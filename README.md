# HACI · HUMAN MARK DOWN

> A sphere of **UD0** — the ROOT0 universe / biosphere. Domain: **ESKIMO BROTHERS**.

HACI · HUMAN MARK DOWN — the first SPECIFIED member of [[aci]] (the H, human-facing): a semantic Markdown profile that records WHO SAID WHAT inside a document's own syntax. Two new prefix symbols + one case convention tag every line by authorial role — ! human command (authority), ? question (lowercase body = AI asks, uppercase / ?! = human asks), > evidence/output, # heading, code fences, and the first-alphabetic-character case (UPPERCASE first letter = Documentation / human prose, lowercase = AI proposal / reasoning). A single-pass, line-by-line, O(n) lexer with code-fence state tracking classifies it — NO NLP, no language model — and it stays valid Markdown in any editor. ⚑ It SELF-HOSTS: the v0.1 spec is written in its own format and parses correctly under its own rules (102 blocks, every role present — verified live). LIVE in-browser parser (type → watch each line attributed, with role stats; load the self-hosting spec). Ships the reference parser hmd.py + the self-hosting spec. Honest: a defensive publication (prior art, NOT a patent); distinct from Markdown extensions (structure not authorship), chat role-fields (which stop at the API boundary), tracked-changes / git-blame (provenance external), and XML/YAML (semantic but not readable Markdown). HACI = Human-AI Collaborative Interchange = Human Mark Down. David Lee Wise / Bridge-Burners LLC.
